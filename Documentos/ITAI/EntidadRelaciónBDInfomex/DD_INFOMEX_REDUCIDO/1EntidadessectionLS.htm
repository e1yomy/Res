CalendarioTipoFecha Esta entidad tiene tres registos predefinidos: 1 - Dia de la semana 2 - Dia del año 3 - Dia Son utilizados cuando configuramos un calendario. - El primero indica que la configuración se aplica a un día de la semana en específico. - El Día del año, es utilizado para señalar días feriados, estos serán contemplados para todos los años. - Día, solamente aplica la configuración para un día en específico.

  GUIDDepartamento Yes No varchar(50) NOT NULL Identificador único de la dependencia o sujeto obligado.
  GUIDDepartamentoPadre No Yes varchar(50) NULL Identificador de del sujeto obligado padre, es decir aquel al que este pertenece.

  Nombre No No varchar(255) NOT NULL Nombre de la dependencia o sujeto obligado.
  Descripcion No No varchar(255) NULL Descripción del sujeto obligado. Este campo se utiliza para describir al sujeto obligado, o bien, escribir información que no es relevante al nombre.
  Ruta No No varchar(7000) NULL Ruta de referencia en cuanto a la relación de ujetos oblijados, por ejemplo: Gobierno del estado/Tesorería
  FechaUltimaModif No No datetime NULL Fecha de la última actualización de lso datos del sujeto obligado.

  Nombre PK FK Tipo de dato Acepta Nulos Descripción
  LimiteInf No No decimal(18,2) NOT NULL Límite inferior en gramos.
  LimiteSup No No decimal(18,2) NOT NULL Límite superior en gramos.
  Zona No No char(1) NOT NULL Indica la zona a la que corresponde el costo de envío. Las zonas son A, B, C.
  Costo No No decimal(18,2) NOT NULL Costo en pesos del servicio de mensajería.

Paso En esta entidad se encuentra la información general de los pasos que sigue un proceso, cada uno de los pasos que se registra en esta tabla, corresponde a una de las figuras del diagrama de proceso construido en visio.
  Nombre PK FK Tipo de dato Acepta Nulos Descripción
  GUIDPaso Yes No varchar(50) NOT NULL Identificador único del paso.
  GUIDProceso No Yes varchar(50) NOT NULL Identificador único de proceso. Este identificador se asigna automáticamente por el sistema al momento iniciar un proceso.
  GUIDProcesoTipoPaso No Yes varchar(50) NOT NULL identificador único del paso de un proceso. Este identificador corresponde a cada uan de las figuras del diagrama del proceso.
  FechaInicio No No datetime NOT NULL Fecha en la que se inicio el paso, esta fecha se da automáticamente por el sistema.
  FechaFin No No datetime NULL Fecha de fin del paso, es la fecha en que se finaliza el paso en el sistema.
  GUIDStatus No Yes varchar(50) NOT NULL Identificador único del estatus.
  FechaUltimaModif No No datetime NULL Fecha de última modificación del registro.
  UsuarioNT No No varchar(50) NULL Nombre del usuario de windows, en caso de contar con este tipo de autenticación.
  FechaVerdeAmarillo No No datetime NULL Fecha en la que se presentará el semáforo amarillo para este paso. Esta fecha se calcula conforme a los días configurados en el diagrama.
  FechaAmarilloRojo No No datetime NULL Fecha en la que se presentará el semáforo rojo para este paso. Esta fecha se calcula conforme a los días configurados en el diagrama.
  FechaAlerta No No datetime NULL Fecha en que se envía una alerta al usuario, siempre y cuando se tengan configuradas alertas en el sistema. la alerta se envía vía correo electrónico.
  FechaCaducidad No No datetime NULL Fecha en la que caduca el paso, esta fecha se calcula conforme a los días configurados en el diagrama.
  GUIDPASOANTERIOR No No varchar(50) NULL Identificador del paso anterior a este.
  GUIDResponsableAnterior No No varchar(50) NULL Identificador del usuario que finalizó el paso anterior.
  FechaFinPasoAnterior No No varchar(50) NULL fecha en la que finalizó el paso anterior.
Proceso Esta entidad contiene la información general de un proceso iniciado en el sistema, es decir, para cada solicitud que se registra en el sistema, se almacena información general en esta tabla (usuario que la registró, fecha de registro, fechas de lso semáforos, fecha de caducidad del proceso, etc.).
  Nombre PK FK Tipo de dato Acepta Nulos Descripción
  GUIDProceso Yes No varchar(50) NOT NULL Identificador único de proceso. Este identificador se asigna automáticamente por el sistema al momento iniciar un proceso.
  GUIDProcesoTipo No Yes varchar(50) NOT NULL Identificador del tipo de proceso al que corresponde.
  Folio No No varchar(255) NOT NULL Folio del proceso. Este folio se asigna automáticamente por el sistema. Es la referencia que conoce el colicitante de sus solicitudes y recursos de revisión.
  GUIDUsuario No Yes varchar(50) NOT NULL Identificador único del usuario.
  FechaInicio No No datetime NOT NULL Fecha en que inicia/captura ela solicitud.
  FechaFin No No datetime NULL Fecha que tendrá como fin el proceso de la solicitud. Según los días configurados para el tipo de proceso.
  FechaAmarillo No No datetime NOT NULL Fecha en que el proceso se presentará con un semáforo de color amarillo. Esto hace refeencia a una alerta.
  FechaRojo No No datetime NOT NULL Fecha en la que el proceso se presenta como tiempo crítico para dar seguimiento. Se presenta en el sistema con un semáforo en rojo.
  GUIDStatus No Yes varchar(50) NOT NULL Identificador único del estatus del proceso. - En proceso - Cancelado - Terminado
  GUIDProcesoPadre No Yes varchar(50) NULL Este campo hace referencia ala misma tabla Proceso con el campo GUIDProceso. Este se utiliza cuando un proceso se dispara a partir de otro. Ejemplo: Subprocesos.
  FechaUltimaModif No No datetime NULL Fecha de última modificación del registro.
  UsuarioNT No No varchar(50) NULL Nombre del usuario de windows, en caso de contar con este tipo de autenticación.
  EstadoPaso No No varchar(50) NULL Estado en el que se encuentra el paso actual del proceso.
  FechaInicioConteo No No datetime NULL Fecha en la que se inicia el conteo de plazos para el proceso iniciado.
  GUIDUsuarioPasoAnt No Yes varchar(50) NULL Identificador del usuario que participó en el paso anterior al paso actual del proceso.
  GUIDPasoAnt No Yes varchar(50) NULL Identificador del paso anterior al actual del proceso.
  FechaPasoAnt No No datetime NULL Fecha en la que se finalizó el paso anterior al actual.
  GUIDPasoPadre No No varchar(50) NULL Identificador del paso en que se disparó un subproceso.
